# Reproducing the paper's experiments

This guide maps every table and figure in the paper *"AI Security Beyond Core
Domains: Resume Screening as a Case Study of Adversarial Vulnerabilities in
Specialized LLM Applications"* to the script that produces it, and gives the
commands to run the full pipeline end to end.

The pipeline has three stages:

1. **Data preparation** – build the 463 job-candidate pairs.
2. **Inference** – run each model under every attack / position / defense
   configuration and write raw classification JSON to `results/`.
3. **Analysis** – aggregate the raw JSON into the paper's tables and figures.

All raw classification results behind the paper's tables ship in this repo, so
the tables can be regenerated without rerunning inference (skip to
[Stage 3](#stage-3-analysis-tables-and-figures)).

The revision result directories are bundled into a single Git LFS archive,
`results/paper_results.tar.gz` (about 124 MB), to keep the repository small.
Fetch and unpack it:

```bash
git lfs install && git lfs pull
bash scripts/extract_results.sh    # -> restores results/revision_* and results/latex_tables/
```

The bundle contains:

- `results/revision_multi_seed/` and `results/revision_multi_seed_aligned/` — open-weight K=5 sweeps (the latter is the manuscript-aligned TP=4 regime).
- `results/revision_multi_seed_aligned_closed/` — closed-source K=5 sweep.
- `results/revision_experiments/` — parser-realism measurements, consensus-subset / per-model CI tables, and human-agreement intermediates (all regenerated by the Stage 3 scripts).
- `results/revision_open_models/` — human-annotation (61-pair) model comparisons.
- `results/latex_tables/` — generated LaTeX tables.

To reduce size, the per-call reasoning traces are kept only for the four
long-trace configurations the covert-rate analysis needs (Qwen3-8B thinking,
gpt-oss-120b low / high, DeepSeek-R1-Distill); for every other file only the
classification labels are retained, which is all the ASR / FRR / agreement
tables read. The single-seed per-model classifications
(`results/results_d_*.json`) and the aggregated CSVs (`results/analysis/`,
`results/utility_analysis/`) are committed directly. Superseded, smoke, and
debugging runs are not included.

---

## 1. Environment

```bash
# analysis + API-client stack
conda create -n resume-attack python=3.10 -y
conda activate resume-attack
pip install -r requirements.txt
```

Scripts pick up the environment name from `CONDA_ENV` (default
`resume-attack`). Heavy, stage-specific components are installed separately:

| Stage | Extra install |
|------|---------------|
| Local open-weight inference | `pip install vllm transformers accelerate peft` |
| FIDS LoRA training | the vendored `LLaMA-Factory/` in its **own** env (`pip install -e LLaMA-Factory`); it pins `numpy<2.0` and must not share the analysis env |
| PDF parser experiments | `PyMuPDF` (already in `requirements.txt`) |

**Secrets.** No keys are committed. Provide them through environment variables;
the shell scripts read these and the Python clients read
`libra_eval/config/api_config.json` (placeholder values committed):

```bash
export HF_TOKEN=...            # gated model downloads
export OPENROUTER_API_KEY=...  # closed models via OpenRouter
export CLOSED_BASE_URL=...     # any OpenAI-compatible gateway for closed models
export CLOSED_API_KEY=...
```

**Hardware.** Open-weight runs in the paper used 4x A800-80GB with
tensor-parallel size 4. 8B models fit on a single 80GB GPU at TP=1.

## 2. Data

The raw datasets and the sampled pairs are tracked with Git LFS:

```bash
git lfs install
git lfs pull
```

This restores:

- `datasets/Linkedin job listings information.json`, `datasets/LinkedIn people profiles_verified_company.json` – the 1,000 jobs / 1,000 profiles.
- `results/job_matching_reverse_150.json` – the **150 jobs sampled with seed 42**, from which up to the top-5 matched applicants per job give the **463 pairs**. This file fixes the sample used throughout the paper.

To regenerate the reverse-matching pool from scratch (not required if the LFS
file is present):

```bash
python dataset_construction/job_matching_reverse.py \
  --output results/job_matching_reverse_150.json --max-matches 150
```

The 463-pair subset is then selected at evaluation time by `--top-k 5
--min-score 0.5` inside `eval/candidate_classifier_vllm.py`.

---

## 3. Evaluation regimes

Two decoding regimes are used, and the analysis scripts expect results produced
under the regime stated in each table caption:

- **Open-weight models** (Qwen3-8B think / non-thinking, Llama-3.1-8B,
  DeepSeek-R1-Distill-Llama-8B, gpt-oss-120b low / high): **K=5 multi-seed**
  (seeds 0–4) at **T=0.7**, tensor-parallel size 4.
- **Closed-source models** (Claude 3.5 Haiku, Gemini 2.5 Flash, GPT-4o-mini,
  GPT-5-mini high / minimal, GPT-5 minimal): **single deterministic decode** at
  **T=0**.

Each (model, attack, position, defense) cell is one configuration. The full
sweep per model is the 2 baselines + 4 attacks x 4 positions x {no-defense,
prompt-defense} = 34 configurations.

## Stage 1+2: generate raw classification results

### Single configuration (smoke test)

```bash
# start a local server
vllm serve Qwen/Qwen3-8B --port 8000 --trust-remote-code --dtype auto

python -m eval.candidate_classifier_vllm \
  --model Qwen/Qwen3-8B --base-url http://localhost:8000/v1 --api-key token \
  --reverse-matches results/job_matching_reverse_150.json \
  --add-adversarial-prompt --adversarial-type job_manipulation \
  --adversarial-position metadata \
  --output results/smoke_job_manipulation_metadata.json
```

Useful flags: `--add-defense-prompt`, `--add-think-parser`,
`--reasoning-effort {low,medium,high}`, `--enable-thinking {true,false}`,
`--temperature`, `--max-tokens`, `--max-prompts` (cap for smoke runs).

### Full open-weight K=5 sweep

`eval/run_multiseed_api.py` runs all 34 configurations x 5 seeds against a
running OpenAI-compatible server. The launcher that reproduces the paper's
open-weight regime (TP=4, seeds 0–4, T=0.7, 463 pairs) for Qwen3-8B
(think/non-thinking), Llama-3.1-8B and DeepSeek is:

```bash
bash scripts/run_multiseed_aligned_tp4.sh
# -> results/revision_multi_seed_aligned/
```

`gpt-oss-120b` (low / high reasoning) uses the same `run_multiseed_api.py`
runner pointed at a gpt-oss server with `--reasoning-effort low|high`.

### Full closed-source sweep

```bash
# set CLOSED_BASE_URL / CLOSED_API_KEY (or OPENROUTER_API_KEY) first
bash scripts/run_closed_multiseed_aligned.sh
# -> results/revision_multi_seed_aligned_closed/
```

For the single-seed T=0 closed runs used in the main tables, drive
`eval/run_closed_model_original_parallel.py` (or `run_multiseed_api.py` with a
single seed and `--temperature 0`). `eval/retry_errored.py` re-runs any cell
that returned `ERROR`.

### FIDS defense (LoRA fine-tuning of Qwen3-8B)

```bash
# train 5 LoRA seeds (own LLaMA-Factory env)
bash scripts/run_fids_multiseed_train.sh     # -> LLaMA-Factory/saves/fids_multiseed/seed{0..4}
# evaluate each trained LoRA (T=0) + seed-0 at T=0.7
bash scripts/run_fids_multiseed_eval.sh
```

The single base SFT recipe is `scripts/run_sft.sh` with
`LLaMA-Factory/examples/train_lora/data_instruction_10k_injected_Qwen_Qwen3-8B_lf-lora-sft.yaml`;
the per-seed configs live in `LLaMA-Factory/examples/train_lora_multiseed/`.
Training data is `datasets/data/train_data/` and is registered for LLaMA-Factory
by `dataset_construction/copy_to_llama_factory.py`.

### Parser-layer (PDF realism) experiments

```bash
bash scripts/run_parser_measured_qwen3.sh    # end-to-end measured ASR, Qwen3-8B non-thinking
```

`eval/parser_realism_experiments.py` builds the hidden-content PDFs and the
naive vs. style-aware extractors; `eval/parser_e2e_measured.py` runs the
end-to-end classification.

## Stage 3: analysis (tables and figures)

To regenerate every table and figure from the shipped results in one go:

```bash
bash scripts/run_all_analysis.sh
```

The rest of this section explains the individual steps that script runs.

First build the aggregated CSV / spreadsheet / per-model LaTeX tables that the
downstream scripts read:

```bash
python eval/analyze_results.py --results-dir results/
# writes results_summary.xlsx, results/analysis/full_results.csv,
#        results/latex_tables/*.tex, and the attack figures
```

Then run the per-table scripts. Pass `--latex` where available to emit
LaTeX-ready rows.

### Master mapping: paper element -> script

| Paper element | Label | Script |
|---|---|---|
| Overall ASR by injection position (per model) | `tab:overall_method_averaged_asr` | `eval/analyze_results.py` -> `eval/analyze_method_averaged_data.py` |
| Overall ASR by attack type (per model) | `tab:position_averaged_overall_asr` | `eval/analyze_results.py` -> `eval/analyze_position_averaged_data.py` |
| Attack-type / position impact figures | `fig:attack_methods_impact`, `fig:attack_positions_impact` | `eval/analyze_results.py` |
| Native-trace covert analysis | `tab:representative_failure_modes` | `eval/why_fail_native.py` |
| Defense by position (Qwen3-8B) | `tab:defense_position_effectiveness` | `eval/qwen3_8b_defense_tables.py` (non-thinking), `eval/qwen3_8b_think_defense_tables.py` (thinking) |
| Defense by attack type (Qwen3-8B) | `tab:defense_method_effectiveness` | same as above |
| Paired defense reduction + bootstrap CI | `tab:defense_reduction_ci` | `eval/qwen3_8b_defense_tables.py` |
| Utility preservation / FRR | `tab:utility_preservation` | `eval/evaluate_utility_preservation.py`, `eval/frr_wilson_ci.py`, `eval/frr_fids_train_multiseed.py` |
| Inter-model agreement (Fleiss/Cohen) | `tab:inter_model_agreement` | `eval/model_consistency_evaluation.py` |
| Per-model attack ASR + Wilson CI | (per-model appendix numbers) | `eval/per_model_wilson_table.py` |
| Human inter-annotator agreement | `tab:human-agreement` | `eval/extended_human_consensus_analysis.py` |
| Attack impact on consensus-NOT_MATCH | `tab:consensus_not_match` | `eval/consensus_subset_aggregate_table.py`, `eval/all_models_consensus_table.py` |
| Domain distribution before/after sampling | `tab:domain_distribution` | `eval/domain_distribution_sampling.py` |
| Parser end-to-end ASR | `tab:parser_e2e` | `eval/parser_e2e_measured.py` -> `eval/parser_e2e_table.py` |
| Evaluation prompt / annotation guide | App. prompt, annotation guide | `eval/candidate_classifier_vllm.py` (prompt), `annotation/` (guide + tool) |

The human-consensus scripts read the three annotator files in `annotation/`;
`eval/extended_human_consensus_analysis.py` derives the 19-pair consensus
`NOT_MATCH` subset that `consensus_subset_aggregate_table.py` and
`all_models_consensus_table.py` consume.

Most analysis scripts write their CSV output under
`results/revision_experiments/`, `results/utility_analysis/`, or
`results/analysis/`.
